* C# create PDF

* use virtual PDF printer

* use C++ create PDF library

* QT QWebView； print HTML content to PDF

* 直接写纯文本的ps文件，然后用命令行转成PDF，
  gpl的单独exe可以用，应该没有license问题。
  需要学习 ps 编程语言。

* 用超文本格式，或者图片格式，然后转换为 PDF
  有 PHP 的 html2pdf 库。

* 如果直接用图片转换，可能因为分辨率问题，维数太大，性能可能有问题。
    使用命令行程序，即使是 GPL 的也可以用，因为不连接，独立进程。

* png2pdf, 命令行程序。windows 版可用编译好的 dklibs->dktools;
    BSD license; 有库


* 直接按PDF文件格式写，应该也可行。可以纯文本，也可以二进制。
  只需要支持贴多个图片即可实现目标，不必支持文本字体、表格等较复杂特性。


PDF规范的发展升级：  
1.1 1995 加入了文档加密（40字节），线索树，名字树，链接，设备独立色彩资源。  
1.2 1996 表单, 半色调屏幕，和其他的一些高级色彩特性, 对中文，日文和韩文的支持  
1.3 2000 数字签名, 逻辑结构， JavaScript, 嵌入式文件，Masked Images, 平滑阴影, 支持 CID字体的附加色彩。  
1.4 2001 文件加密 (128 字节), 标签式 PDF, 访问控制，透明，元数据流  
1.5 2003 文档加密 (公钥), JPEG 2000 压缩, 可选的内容组，附加的注解类型  
1.6 2005 文档加密 (AES),增加最大文件支持，加入3D支持，额外的注解类型
