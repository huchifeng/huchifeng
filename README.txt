mygit 的主要 idea:

1. 整个仓库一个文件
从 CVS,SVN 发展到 GIT 等分布式版本控制库，
仓库从每个目录下一个 CVS 目录或 .svn 目录，变为整个工作目录的根目录下只有一个 .git；
下一步仓库应该演变为整个仓库一个文件，更为方便。

2. 使用64位整数作为仓库中对象的id
使用了hash值，但仅用于快速的排除重复内容，而不作为对象的id。

3. 仅提供一个最简单的存储层，可用于文件备份，
同样内容反复添加对库没有影响，
添加相同内容但不同文件名的文件，仅记录新的符号名。
可以添加相同文件名不同内容的文件，
所以应该也可以用于版本控制，需要编写脚本或其他语言的程序，以实现整个目录的捡入和检出。

4. 每个文件都是完整的存入，不提供差分存储等高级功能。

目前仓库中包括两种记录
1. 内容： id+hash值+文件内容
2. 符号名：id+所引用的内容对象的id+符号名+建立日期
用于文件备份已经可以了。
用于项目的版本控制，可能需要添加用于目录的对象，以及用于提交注释的对象。
还需要开发仓库之间的内容交换功能。
