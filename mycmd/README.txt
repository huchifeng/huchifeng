mycmd 的主要 idea:

1. 使用 cmdlet 编程，cmdlet 可能是一个 exe，
或者一个 dll，
或者一个本地 service，
或者一个 web service；
等等。
类似 unix 的过滤器程序，
不基于文本行
命令之间使用类似 json 的串化格式传递数据，但要支持二进制数据。
格式定义严格，无歧义，输出的数据必须合规，
最后给用户的输出才由 mycmd 自动用 table 或其他方式显示。
使用模拟的文件系统传递共享数据。

解决 unicode 等问题；

2. 使用最简单的语法，不用特殊符号，
[a-z0-9+-.] 用于表达标识符和科学计数法是必须的。
不用大写字母，不用 * / % { } [ ] ( )
